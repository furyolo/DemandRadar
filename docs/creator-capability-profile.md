# Creator Capability Profile

本文件用于 DemandRadar 的供给侧判断：每次挖掘到需求后，先评估个人能力能否直接满足，再评估 AI Agent 能否短期补足，最后才评估第三方供给路径。

来源参考：`D:\Coding\StoryForge\docs\creator-persona.md`。本文件只抽取与 DemandRadar 产品机会履约相关的能力，不继承影视内容创作定位。

## 可直接复用的个人能力

- AI 应用、自动化脚本、API 编排、前后端系统和工具化交付。
- 数据分析、SQL、广告 ROI、用户行为、游戏数据、指标体系和预测判断。
- 平台机制、商业模式、供需匹配、交易路径和现有供给缺口识别。
- 结构化研究、信息整理、报告生成、MVP 原型和工作流产品化。
- 游戏、内容平台、增长优化和商业化机制观察。

## 判断顺序

1. `个人可直接满足`：需求落在 AI 自动化、数据分析、流程产品化、平台机制或结构化研究内，并且不依赖牌照、线下履约或强专业责任。
2. `AI Agent 短期补足`：个人可以负责判断、产品化和交付边界，AI Agent 可承担接单、分诊、资料整理、比较、生成、监控、路由和协调。
3. `第三方供给优先`：需求涉及医疗、法律、会计、监管牌照、线下上门、仓储、制造、物流、硬件或其它高责任履约。

## 使用边界

- 个人能力匹配是机会评估，不是交付承诺。
- AI Agent 可补足的是结构化流程和执行容量，不自动代表需求已可商业交付。
- 现有供给判断仍必须来自 `current_alternatives` 或 `competitor` 类型市场证据。
- 第三方供给不足时，DemandRadar 应把缺口暴露为下一轮检索方向。
