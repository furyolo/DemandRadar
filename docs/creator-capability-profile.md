# Creator Capability Profile

本文件用于 DemandRadar 的供给侧判断：每次挖掘到需求后，先评估个人能否自营交付并获得主要利润；如果不能，再评估个人 + AI Agent 能否补足能力缺口并继续自营；仍不能时，才评估外部供给撮合路径。

来源参考：`D:\Coding\StoryForge\docs\creator-persona.md`。本文件只抽取与 DemandRadar 产品机会履约相关的能力，不继承影视内容创作定位。

## 可直接复用的个人能力

- AI 应用、自动化脚本、API 编排、前后端系统和工具化交付。
- 数据分析、SQL、广告 ROI、用户行为、游戏数据、指标体系和预测判断。
- 平台机制、商业模式、供需匹配、交易路径和现有供给缺口识别。
- 结构化研究、信息整理、报告生成、MVP 原型和工作流产品化。
- 游戏、内容平台、增长优化和商业化机制观察。
- 硕士研究生学历，985/211 大学教育背景；可支撑结构化研究、资料筛选、学术/技术信息理解、学习型产品和教育相关需求的前期判断。

## 判断顺序

1. `个人自营交付`：需求落在 AI 自动化、数据分析、流程产品化、平台机制或结构化研究内，并且不依赖牌照、线下履约或强专业责任；利润主要归个人。
2. `AI Agent 增强自营`：个人可以负责判断、产品化和交付边界，AI Agent 可承担接单、分诊、资料整理、比较、生成、监控、路由和协调；成本主要是工具/API 和人工复核。
3. `外部供给撮合`：需求涉及医疗、法律、会计、监管牌照、线下上门、仓储、制造、物流、硬件或其它高责任履约；个人主要做需求澄清、供给筛选和交易撮合，赚取中介费或差价。

## 使用边界

- 个人自营匹配是机会评估，不是交付承诺。
- 学历背景可作为研究能力、可信背书和学习型场景判断的加分项，但不等同于教师资质、行业牌照或线下履约能力。
- AI Agent 可补足的是结构化流程和执行容量，不自动代表需求已可商业交付。
- 外部供给撮合判断仍必须来自 `current_alternatives` 或 `competitor` 类型市场证据。
- 外部供给不足时，DemandRadar 应把缺口暴露为下一轮检索方向。
